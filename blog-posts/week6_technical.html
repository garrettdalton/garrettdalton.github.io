Classes vs. Modules

Class is actually a subclass of Module

Classes and Modules are very similar at first look. Both are made up of different methods and constants, and can both use instance methods. Here is how the two look when coded:

As you can see, the two are very similar. The only difference is using the key word module instead of class.

So what's the difference?

Ok. So class and module at first glance look similar, and share similar attributes. The difference however, is the programmers intended use for them. Classes are used for the instantiation of objects and the corresponding instance methods. Modules are all about methods. Modules can also include instance methods, or more broader module methods, but cannotbe instantiate as an object. They also cannot inherit or be inherited from like classes. A big difference for modules, is that they can be included within classes.

This inclusion is a big reason to use a module.

Because you can include modules within a class, and you can use theses inclusions over multiple classes, you can reuse method code across multiple classes with a simple include call. This almost makes a module a small 'library' of methods which classes can access using include. Let's go back to the earlier example, this time with the class including the module.
