SQL Injection Attacks

SQL injection attacks are a somewhat simple and common method of maliciously accessing and effecting an SQL database. It is generally used where a web app accepts user input in order to acces a SQL database.The attacker will use their input to trick the database into using certain commands in SQL, which in turn can give the user full access, or allow the user to modify data at their leasure.

The method is simple enough in that an attacker can give input which has a hidden command within it. In the case of the XKCD comic, the input "Robert'); DROP TABLE students; --" closes the the statement by ending Robert with ');. This will give the input as Robert and the semi colon begins a new command input. 'DROP TABLE students -- ' then deletes the entire student table (if it is named students) and '--' then makes the rest of the line a comment. In this case the attacker could most like guess that the table is named students, assuming that the school has a simple database. In the real world, attackers will prob databases to try to find weaknesses and table and column names in order to access the database.

Is there any way to prevent such an attack?

There are two relatively simple techniques for new users to protect from injection attacks. The first is to limit the user permissions of all but the moderators of the database. Setting a users permission to query only will limit them from modifying data in the first place.

The second method is mentioned in the comic. As a database moderator, you should strive to sanitize the input, which is a method to filter input from symbols and commands frequently used in injection attacks. Characters like the semi-colon can be filtered from input to avoid an attacker from injecting commands. Obviously this filter depends on what the input is requesting, and the sanitizing should reflect the data being input.
